# CLAUDE.md — wtml

## Project Overview

**wtml** is a new ML-family programming language implemented in OCaml. It is a strict (eager) functional language with type inference, a simple module system, no OOP, and a novel effect/side-effect tracking system integrated into the type system. The compiler targets a Lisp VM.

The project is in early design/bootstrap phase. The language specification lives in `memo.txt` (written in Japanese).

## Language Design (wtml)

The language specification is in `memo.txt`. Key design decisions:

### Core Properties
- **Functional language** — ML family, no OOP
- **Strict (eager) evaluation**
- **Type inference** with explicit annotations optional
- **Simple module system** — `module Name`, access via `Module.value`, `import` to open
- **Side effects tracked in the type system**

### Syntax Summary

| Construct | Syntax |
|---|---|
| Value binding | `let x = expr` |
| Function definition | `fn name args = body` |
| Anonymous function | `fn x -> expr` |
| Type annotation | `expr : type` |
| Block (multi-expr) | `{ ... }` |
| Pattern match | `match expr { \| pat => expr ... }` |
| Function with match body | `fn name = { \| pat => expr ... }` |
| Module definition | `module Name` |
| Module import | `import Name` |
| Module access | `Module.value` |

### Side-Effect System

Functions use naming suffixes and arrow types to track side effects:

| Suffix | Meaning | Arrow Type |
|---|---|---|
| (none) | Pure, no side effects | `->` |
| `!` | Has side effects | `->!` |
| `~` | Effect-polymorphic (depends on arguments) | `->~` |

**Subtyping rules:**
- `->` <: `->!` — pure functions can be used where effectful ones are expected
- `->` <: `->~` — pure resolves `~` to pure
- `->!` <: `->~` — effectful resolves `~` to effectful
- `->!` cannot be used where `->` (pure) is expected — compile error

### Inference Rules
- Body calls a `!` function → the function itself must be `!`
- Body only passes `~` arguments through → can be `~`
- No effects → pure (`->`)

## Development Conventions

### Language
- Commit messages are in **Japanese** (following existing history) or English — either is acceptable.
- Design documents (`memo.txt`) are in Japanese.

### Code Style
- Follow standard OCaml conventions (snake_case for values, PascalCase for modules/types).
- Keep the compiler implementation in `lib/`.
- Tests go in `test/test_wtml.ml`.

### Git Workflow
- `main` is the default branch.
- Feature branches use the `claude/` prefix for AI-assisted work.
- `.gitignore` excludes `_build/` and `bin/`.

### Adding New Source Files
- Place new `.ml` / `.mli` files in `lib/`. Dune auto-discovers them.
- If adding dependencies, update the `(depends ...)` field in `dune-project` and run `dune build` to regenerate `wtml.opam`.

### Testing
- Run tests with `dune test`.
- Add test cases to `test/test_wtml.ml`.
- No test framework is configured yet — when adding one, consider `alcotest` or `ppx_inline_test`.
